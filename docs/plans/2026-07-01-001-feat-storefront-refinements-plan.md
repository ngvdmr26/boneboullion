# feat: BONE BOUILLON storefront refinements + image asset pipeline

## Summary

Полировочная итерация по витрине BONE BOUILLON (репозиторий `mamheg/BoneBoullion`, ветка `main`) плюс подготовка ассет-пайплайна для реальных изображений. Две дорожки:

- **A. UI-переработка** существующих компонентов: убрать верхнюю жёлтую плашку доставки; сменить серифный шрифт на **Spectral** и увести цены/цифры на **Manrope tabular**; заменить блок «скидка 10%» на подписку в **Telegram**; кнопка корзины ведёт на страницу `/cart` (drawer убираем); новая анимация поля поиска; убрать соцдоказательство с аватарками; иконки преимуществ — белые в жёлтом круге; добавить **карточку установки веб-приложения на экран** (PWA install prompt, как в AmoreSample).
- **B. Ассет-пайплайн**: по скриншотам Instagram собрать **PDF** с описанием реальных товаров и написать **один master-промпт для nano-banana**, который сгенерит все картинки сайта (лого, hero, карточки). Внедрение сгенерированных картинок в код — отдельный отложенный шаг (зависит от готовых ассетов).

Решения, зафиксированные с пользователем: сериф — **Spectral**; цены — **Manrope tabular-nums**; контент Instagram — **пользователь загрузит скриншоты**; бренд на лого — **BONE BOUILLON**.

## Problem Frame

Текущая витрина (собрана в предыдущей итерации) выглядит «чуть слабовато»: цифры цен в Playfair смотрятся криво, сериф хочется свежее, верхняя жёлтая плашка и промо «скидка 10%» не нужны, соцдоказательство с кругляшками лишнее, корзина открывается сбоку вместо перехода на страницу, поле поиска статичное, а товарные картинки — SVG-плейсхолдеры вместо реальных фото. Нужно поднять ощущение «дороже» и подготовить реальные изображения.

## Key Technical Decisions

- **Шрифты.** `--font-display` → **Spectral** (через `@fontsource/spectral`, кириллица + латиница). Заголовки и названия товаров остаются серифом (Spectral). **Цены и любые денежные числа** уходят с `font-display` на `font-sans` (**Manrope**) + `font-variant-numeric: tabular-nums` — это чинит «кривые» цифры и убирает прыжки ширины. Playfair Display удаляется из зависимостей и импортов.
- **Корзина — только страница.** Drawer убираем полностью: проще модель, один источник правды (`/cart`). Удаляем `CartDrawer`, состояние `isCartOpen/openCart/closeCart` из `UIContext` (оставляем только мобильное меню). Кнопки корзины в хедере становятся `Link to="/cart"`.
- **Анимация поиска** — на `framer-motion` с shared-layout (`layoutId`), т.к. библиотека уже в проекте. Пустое поле → только иконка-лупа слева, без кнопки. При вводе иконка «перекатывается» вправо и превращается в жёлтую круглую кнопку submit; на ховере кнопка расширяется и показывает белую надпись «Найти». Reduced-motion фолбэк: без переката, кнопка просто появляется справа.
- **Иконки преимуществ.** «Белая иконка в жёлтом круге». Чтобы белое читалось по AA, круг — насыщенно-золотой (`brand-500`/`brand-600`), а не светло-жёлтый `brand-400` (белое на нём проваливается по контрасту). Финальный оттенок подбираем на этапе выполнения, дефолт — `brand-600` с белой иконкой.
- **Instagram недоступен автоматизации** (логин-стена) → источник данных = скриншоты от пользователя. План не полагается на веб-скрейпинг.
- **Ассет-пайплайн — один master-промпт.** Промпт для nano-banana перечисляет весь инвентарь картинок (лого, hero, по одной на каждый SKU), с единым визуальным стилем, форматами/пропорциями под слоты сайта, и списком референсов (скриншоты IG + текущий лого + бренд-цвет `#E69E26`). Цель — реалистичная предметная фуд-фотография.

### U5. Анимация поля поиска (иконка «перекатывается» вправо → жёлтая кнопка «Найти»)

**Goal:** Пустое поле — только иконка-лупа слева, без жёлтой кнопки. При вводе иконка плавно уезжает вправо и становится жёлтой круглой кнопкой submit; на ховере кнопка расширяется и показывает белую надпись «Найти».
**Files:** `src/components/ui/SearchBar.tsx` (переписать с `framer-motion`).
**Approach:** Один общий motion-элемент лупы с `layoutId` (или `layout`), позиционируемый слева при пустом вводе (декоративно, `pointer-events-none`) и справа как кнопка при непустом — framer анимирует переход позиции («перекат»). Жёлтый фон (`brand-600`) появляется при переходе в состояние кнопки. Надпись «Найти» — `AnimatePresence`, белый текст, разворачивается по ширине на `hover`/`focus` кнопки (scale/opacity/width или auto-layout). Submit по Enter работает в обоих состояниях. Reduced-motion: без переката — кнопка просто появляется справа (мгновенно), лупа слева гаснет.
**Technical design (directional):**
```
state empty:   [🔍 ................................]   (иконка слева, submit скрыт)
state typing:  [ ...........................  (🔍)]   (иконка «перекатилась» вправо, круг жёлтый)
hover/focus:   [ .......................  (🔍 Найти)] (кнопка расширилась, белый лейбл)
```
Переход позиции лупы — через shared layout; ширина кнопки и лейбл — через layout-анимацию с `spring, bounce: 0`.
**Patterns to follow:** motion-паттерны из `src/components/cart/CartDrawer.tsx` (до удаления) и `src/components/catalog/ProductCard.tsx` (иконка-swap с `AnimatePresence`, `spring duration 0.3 bounce 0`); принципы `make-interfaces-feel-better` (scale/opacity/blur, `transition` только по конкретным свойствам, hit-area ≥ 40px у кнопки).
**Test scenarios (manual, нет тест-раннера):** (1) пустое поле — кнопки нет, лупа слева; (2) ввод символа — лупа уезжает вправо, круг желтеет; (3) ховер/фокус кнопки — появляется белый «Найти»; (4) Enter отправляет на `/catalog?q=…`; (5) очистка поля — возврат в исходное состояние без дёрганья; (6) `prefers-reduced-motion` — переход мгновенный, без переката.
**Test expectation:** none — нет тест-харнеса в проекте; верификация ручная + сборка.
**Verification:** поведение по сценариям 1–6 в `npm run dev`; сборка зелёная.

### U10. Карточка установки веб-приложения на экран (PWA install prompt)

**Goal:** Всплывающая ненавязчивая карточка «Установить BONE BOUILLON на экран», как в AmoreSample: на Android/Chrome запускает нативный install-промпт, на iOS/браузерах без него — показывает пошаговую инструкцию.
**Files (создать):** `src/components/pwa/InstallPrompt.tsx` (карточка + логика `beforeinstallprompt`), `src/components/pwa/IosInstallModal.tsx` (ручная инструкция по браузерам), подключить в `src/components/layout/Layout.tsx`.
**Approach:**
- Слушать `window` событие `beforeinstallprompt`, `preventDefault()`, сохранить событие; показать карточку снизу (над мобильной таб-навигацией) с CTA «Установить». По клику — вызвать `prompt()` у сохранённого события; по `appinstalled` или установке — скрыть.
- **Скрывать карточку**, если приложение уже запущено как PWA: `window.matchMedia('(display-mode: standalone)').matches` или `navigator.standalone` (iOS).
- **iOS / браузеры без `beforeinstallprompt`** (Safari): нативного события нет — карточка ведёт на `IosInstallModal` с шагами «Поделиться → На экран „Домой“» (детект iOS/Safari; иконка Share из `lucide-react`).
- **Dismiss с памятью:** крестик закрывает карточку и пишет флаг в `localStorage` (напр. `bb_install_dismissed`), чтобы не надоедать; не показывать повторно N дней (дефолт — пока флаг стоит).
- Появление — с задержкой/после первого скролла, мягкая анимация въезда (`framer-motion`, `spring bounce:0`), стиль — бренд-карточка (`bg-card`, `shadow-pop`, скругление `rounded-card`), иконка/лого BONE BOUILLON.
**Technical design (directional):**
```
beforeinstallprompt (Chrome/Android) → save event → показать карточку
  ▸ «Установить» → savedEvent.prompt() → appinstalled → скрыть + флаг
iOS Safari (нет события) → карточка «Как установить» → IosInstallModal (шаги Share→На экран)
already standalone → ничего не показывать
dismiss (✕) → localStorage flag → не показывать
```
**Patterns to follow:** AmoreSample `src/components/InstallHint.tsx` и `src/components/IosInstallModal.tsx` (детект standalone, тип `BeforeInstallPromptEvent`, набор per-browser инструкций, тексты «На экран „Домой“»); PWA-манифест уже настроен в `vite.config.ts` (`VitePWA`), тема `#E69E26`; motion-паттерны проекта (`spring, bounce: 0`).
